Welcome to Location Logga

I made this just to play around a bit with cloud computing.

Also, I just moved and I thought, instead of checking google maps all the time, I'd just automate that

This project costs me about $20 USD a month to maintain.

Repo and source code available here: link

Tech used:

  • Frontend: HTML/CSS/JS, with Materialize (for prettying the view), Chart.js (for basic dashboards)
  • PostGres (hosted on GCP) for persisting segment data to DB (yes, it's overkill and was the main cost)
  • Google's Distance Matrix API for getting real time location
  • Google's Cloud Functions for invoking the distance API, and inserting/retrieving data from PostGres
  • Google's Cloud Scheduler for scheduling cloud functions for execution
  • Google's Pub/Sub for help/convenience in turning on/off the DB to save on cost

Current available segment dashboards:

  • oxnard-ucsb
  • oxnard-ktown
  • oxnard-sawtelle

What did I learn? What would I improve?

  • Section in progress.
  • Pain point - my DB was always turned off. It was super cheap. it took forever to turn on. I'd probably trade up for a more expensive DB if I could ensure turning on/off was under a minute. Sometimes, it would take up to 10-20 minutes to turn on / off.
